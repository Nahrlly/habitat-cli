# OpenClaw Habitat Orders

For each scheduled inspection, work in the Habitat CLI workspace and perform exactly one cycle:

1. Read current Habitat humans, EVA status, and world sector state through the local Habitat API.
2. Run `habitat autonomy run-now` with command execution available in the session.
3. Let the autonomy policy reject unsafe actions before selecting an action.
4. Allow at most one deployment, movement, collection, or no-op per cycle.
5. Keep the audit record generated by the CLI. Do not edit SQLite state manually.
6. Return the concise CLI result.

OpenClaw cron should use the user-selected schedule and a variable-driven Discord destination:

```sh
openclaw cron add \
  --name "Habitat Autonomy Inspector" \
  --every 5m \
  --session isolated \
  --agent main \
  --message "Work in the Habitat CLI workspace. Read ORDERS.md and perform exactly one habitat autonomy run-now cycle. Return the concise result." \
  --tools exec,read,write \
  --announce \
  --channel discord \
  --to "channel:$DISCORD_CHANNEL_ID" \
  --timeout-seconds 120
```

Never place tokens, device codes, credentials, private addresses, or channel IDs in this file.
